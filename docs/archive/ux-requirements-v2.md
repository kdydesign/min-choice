# 베베 초이스 UX 요구사항 정리 v2

## 문서 목적

이 문서는 과거 작업 중 정리한 UX 요구사항 메모입니다.  
현재는 **참고용 문서**이며, 최종 UX 기준은 [../ux-spec.md](../ux-spec.md)를 따릅니다.

## 이 문서의 현재 역할

- 주요 요구 변경 이력 정리
- 화면 개선 요청의 원문 보관
- 최종 UX 스펙 작성 시 참고 자료

## 요약된 요구사항

### 1. 서비스 방향

- 모바일 우선
- 카드형 구조
- 귀엽지만 과하지 않은 육아 앱 톤
- 알레르기 경고 우선

### 2. 정보 구조

- 로그인
- 아이 프로필
- 오늘 식단
- 최근 식단

### 3. 로그인 페이지 요구

- 불필요한 문구 제거
- 중앙 브랜드 강조
- 로그인 방식 버튼 중심
- 하단 카피라이트
- 공통 블러 배경

### 4. 아이 프로필 페이지 요구

- 아이 선택이 쉬워야 함
- 카드형 리스트 중심
- 프로필 등록과 수정이 명확해야 함

### 5. 오늘 식단 페이지 요구

- 끼니별 재료 입력이 핵심
- 선택된 아이 정보가 상단에 보여야 함
- 결과 화면과 입력 화면의 역할이 분리되어야 함

### 6. 최근 식단 페이지 요구

- 이력은 날짜 그룹 기준으로 스캔 가능해야 함
- 불러오기 흐름이 쉬워야 함

### 7. 생성 진행 상태 요구

- 생성 중 상태가 보여야 함
- 진행 상태는 실제 단계 기반이어야 함
- 생성 중에는 화면 상호작용을 막아야 함

### 8. 브랜딩 / 비주얼 요구

- 베베 초이스 브랜딩 사용
- 파스텔 톤
- 부드러운 일러스트와 둥근 카드

### 9. 모바일 사용성 요구

- 한 손 조작
- 큰 버튼
- 짧은 입력 흐름
