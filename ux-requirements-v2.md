# Bebe Choice UX Requirements

## 1. 서비스 방향

- 앱 이름은 `베베 초이스`이며 영어 표기는 `Bebe Choice`를 사용한다.
- 현재 인증 운영 경로는 `Google 로그인`과 `익명 사용`만 지원한다.
- `Kakao 로그인`은 후속 작업으로 보류한다.
- 모바일 우선 사용성을 기준으로 화면과 흐름을 설계한다.

## 2. 정보 구조

- 앱은 페이지 단위로 구분한다.
- 최소 페이지 구성은 아래와 같다.
  - `오늘 식단`
  - `최근 식단`
  - `아이 프로필`
- 하단 네비게이션을 통해 주요 페이지를 이동할 수 있어야 한다.

## 3. 로그인 페이지

- 불필요한 설명 문구는 제거한다.
- 화면 중앙에 `베베 초이스` 타이틀을 배치한다.
- 가능하면 로고를 함께 보여준다.
- 타이틀 하단에는 로그인 방식 버튼만 배치한다.
  - `Google로 로그인`
  - `익명으로 계속하기`
- 화면 맨 아래에는 `mindoong-corp` 카피라이트를 배치한다.
- 로그인 페이지 배경은 블러 처리된 이미지형 디자인을 사용한다.
- 이 배경 방향은 모든 페이지에서 공통으로 유지한다.

## 4. 아이 프로필 페이지

- 최초 로그인 후 프로필이 하나도 없으면 `아이 프로필` 페이지가 먼저 열려야 한다.
- 최초 프로필은 이 페이지에서 등록한다.
- 이후에도 같은 페이지에서 수정과 삭제가 가능해야 한다.
- 여러 아이가 있을 경우 빠르게 전환할 수 있어야 한다.

## 5. 오늘 식단 페이지

- 메인 화면은 `오늘의 추천 식단`을 중심으로 보여준다.
- 끼니별 재료 입력은 같은 화면에서 가능해야 한다.
- 다만 입력 영역은 메인 결과보다 덜 무겁게, 더 간결하게 보여준다.
- 입력 재료가 바뀌면 이전 추천 식단을 그대로 보여주지 않는다.
- 새 입력이 생긴 경우 `다시 생성이 필요하다`는 상태를 명확하게 보여줘야 한다.

## 6. 최근 식단 페이지

- 최근 식단은 메인 화면 내부 보조 섹션이 아니라 네비게이션 가능한 별도 페이지로 보여준다.
- 저장된 식단 이력을 목록으로 확인할 수 있어야 한다.
- 특정 이력을 다시 열어 결과를 확인할 수 있어야 한다.

## 7. 생성 진행 상태

- 식단 생성 중 로딩 상태를 사용자에게 명확히 보여줘야 한다.
- 단순 스피너보다 단계형 진행 상태가 바람직하다.
- 예시:
  - 재료 정리 중
  - 추천 식단 생성 중
  - 저장 중

## 8. 브랜딩/비주얼

- 앱 전체 디자인은 아이 서비스에 맞게 귀엽고 부드러운 인상을 가져야 한다.
- 과도하게 복잡하기보다 친근하고 따뜻한 톤이 우선이다.
- 파스텔 계열, 둥근 카드, 부드러운 인터랙션을 기본 방향으로 한다.
- 타이포와 컴포넌트는 브랜드 일관성을 유지해야 한다.

## 9. 모바일 사용성

- 모바일이 주 사용 환경이다.
- 주요 액션 버튼은 엄지 범위 안에서 누르기 쉬워야 한다.
- 하단 네비게이션, 카드 간격, 입력 필드 크기, 버튼 높이는 모바일 기준으로 최적화한다.
- 작은 화면에서도 스크롤 흐름이 자연스럽고 정보 우선순위가 명확해야 한다.

## 10. 현재 반영되어야 하는 핵심 동작

- Google 로그인과 익명 사용은 모두 유지한다.
- 익명 상태에서 사용한 데이터는 로그인 후 이어받을 수 있어야 한다.
- 입력 재료와 저장된 추천 결과의 불일치 상태를 감지해야 한다.
- 식단 생성 중 진행 상태를 보여줘야 한다.
- 배경 비주얼은 전 페이지 공통으로 유지해야 한다.
